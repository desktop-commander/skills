# Pull Request Template

## 概要

一貫性のあるPRを作成するためのテンプレートとガイドライン。

## 基本テンプレート

```markdown
## 概要

[このPRの目的を1-2文で説明]

## 変更内容

- [変更点1]
- [変更点2]
- [変更点3]

## 関連Issue

Closes #[Issue番号]

## テスト

- [ ] 単体テストを追加
- [ ] 手動テストを実施
- [ ] 既存テストがパス

## スクリーンショット

[UI変更がある場合のみ]

## 確認事項

- [ ] コードレビューの準備完了
- [ ] ドキュメントを更新（必要な場合）
- [ ] 破壊的変更なし（または記載済み）
```

## PRタイプ別テンプレート

### 機能追加（feat）

```markdown
## 概要

[新機能の説明]

## 変更内容

### 追加
- [新機能の詳細]

### 変更
- [既存機能への影響]

## 動作確認

1. [手順1]
2. [手順2]
3. [期待する結果]

## テスト結果

```
[テスト実行結果]
```

## 関連Issue

Closes #[番号]
```

### バグ修正（fix）

```markdown
## 概要

[バグの説明と修正内容]

## 原因

[バグの原因]

## 修正内容

- [修正方法]

## 再現手順

1. [手順]
2. [手順]
3. [バグが発生]

## 修正後の動作

1. [手順]
2. [手順]
3. [正常に動作]

## 関連Issue

Fixes #[番号]
```

### リファクタリング（refactor）

```markdown
## 概要

[リファクタリングの目的]

## 変更内容

### Before
[変更前のコード構造]

### After
[変更後のコード構造]

## 影響範囲

- [影響するファイル・機能]
- [既存動作への影響なしを確認]

## テスト

- [ ] 既存テストがパス
- [ ] 動作に変更なし
```

## 自動生成ルール

### タイトル生成

```
[プレフィックス]: [簡潔な説明]
```

**例**:
- `feat: AI人格エディター機能を追加`
- `fix: ログインエラーを修正`
- `refactor: エージェントクラスを統合`

### 本文生成

**コミットから自動抽出**:

```bash
git log [base]..HEAD --pretty=format:"- %s"
```

**結果例**:
```
## 変更内容

- feat: エージェント基本機能を追加
- fix: エラーハンドリングを修正
- test: エージェントテストを追加
```

## PR作成フロー

### Step 1: ベースブランチ確認

```
現在のブランチ: feature/new-ai
ベースブランチ: main

→ 正しいですか？ [y/n]
```

### Step 2: タイトル入力

```
提案タイトル: feat: AI機能の追加

→ このタイトルでよいですか？ [y/edit]
```

### Step 3: 本文入力

```
変更内容（自動生成）:
- agent_a.py: 新規追加
- agent_b.py: 新規追加
- main.py: AI機能統合

→ 追加の説明を入力しますか？ [y/n]
```

### Step 4: ラベル・レビュアー

```
ラベル: enhancement
レビュアー: [指定なし]

→ これでPRを作成しますか？ [y/n]
```

## レビュー依頼メッセージ

PR作成後に自動生成可能：

```markdown
## レビュー依頼

@reviewer 

以下のPRのレビューをお願いします：
- [PR URL]

### 重点確認事項
- [特に見てほしい点]

### 確認期限
[期限がある場合]
```

## よくない例

### ❌ 説明不足

```
修正しました
```

→ ✅ 何をどう修正したか明記

### ❌ 変更範囲が広すぎる

```
100ファイル変更、5000行追加
```

→ ✅ 適切なサイズに分割

### ❌ テストがない

```
テスト未実装
```

→ ✅ 最低限のテストを追加

## マージ後のクリーンアップ

PRマージ後の推奨アクション：

```bash
# ブランチ削除
git branch -d feature/xxx
git push origin --delete feature/xxx

# 最新を取得
git checkout main
git pull origin main
```
