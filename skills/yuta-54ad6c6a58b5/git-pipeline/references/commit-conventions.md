# Commit Message Conventions

## 概要

一貫性のあるコミットメッセージ規約に従うことで、変更履歴の可読性と保守性を向上させる。

## フォーマット

```
<プレフィックス>: <簡潔な説明>

[オプション: 詳細説明]
[オプション: フッター]
```

## プレフィックス

### 主要プレフィックス

| プレフィックス | 意味 | 例 |
|--------------|------|-----|
| `feat:` | 新機能追加 | feat: ユーザー認証機能を追加 |
| `fix:` | バグ修正 | fix: ログインエラーを修正 |
| `docs:` | ドキュメント更新 | docs: READMEを更新 |
| `style:` | コードスタイル変更（機能に影響なし） | style: インデントを修正 |
| `refactor:` | リファクタリング | refactor: 認証ロジックを簡素化 |
| `test:` | テスト追加・修正 | test: 認証テストを追加 |
| `chore:` | その他の変更 | chore: 依存関係を更新 |
| `perf:` | パフォーマンス改善 | perf: クエリを最適化 |
| `ci:` | CI設定変更 | ci: GitHub Actionsを設定 |
| `build:` | ビルドシステム変更 | build: webpack設定を更新 |

---

## メッセージ例

### feat（新機能）

```
feat: AI人格エディター機能を追加

- 人格プロファイルの作成・編集をサポート
- リアルタイムプレビューを実装
- エクスポート機能を追加

Closes #42
```

### fix（バグ修正）

```
fix: ログイン時のセッションエラーを修正

トークン有効期限チェックの条件が逆になっていた問題を修正。

Fixes #123
```

### refactor（リファクタリング）

```
refactor: エージェントクラスを統合

AgentA, AgentBの共通部分をBaseAgentクラスに抽出。
重複コードを削減し、保守性を向上。
```

### docs（ドキュメント）

```
docs: インストール手順を更新

Python 3.11対応の手順を追加。
```

---

## 作成ルール

### 簡潔な説明（1行目）

- **50文字以内**を目指す
- **動詞で始める**（追加、修正、更新など）
- **主語を省略**（「xxした」ではなく「xxを追加」）
- **句点（。）を使わない**

### 詳細説明（オプション）

- **何を変更したか**ではなく**なぜ変更したか**を書く
- 改行で区切る
- 各行72文字以内

### フッター（オプション）

```
Closes #42        // Issueを閉じる
Fixes #123        // Bug Issueを閉じる
Related #45       // 関連Issue
Breaking change: // 破壊的変更の通知
```

## 自動生成の判断基準

### 変更内容からプレフィックスを推測

| 変更パターン | プレフィックス |
|------------|--------------|
| 新しいファイル追加 | feat |
| 関数・クラスの追加 | feat |
| バグ修正コメントあり | fix |
| エラー処理の追加 | fix |
| コメント・README更新 | docs |
| テストファイルの変更 | test |
| 変数名・関数名変更 | refactor |
| ファイル移動・削除 | refactor |
| 設定ファイル変更 | chore |
| パッケージ更新 | chore |

### 複数の変更がある場合

**優先順位**:
1. feat / fix（機能・バグ関連）
2. refactor
3. test
4. docs
5. chore

**例**:
- 新機能 + テスト追加 → `feat:`
- リファクタリング + ドキュメント更新 → `refactor:`

## よくない例

### ❌ 情報が不足

```
修正
```

### ❌ 主語が含まれる

```
ユーザーがログインできるようにしました
```

→ ✅ `feat: ユーザーログイン機能を追加`

### ❌ 複数の変更が混在

```
feat: 新機能追加、バグ修正、ドキュメント更新
```

→ ✅ 別々のコミットに分割

### ❌ 行が長すぎる

```
feat: AI人格エディター機能を追加して、リアルタイムプレビューを実装し、エクスポート機能も追加した
```

→ ✅ 詳細は本文に書く

## ツール連携

### 自動生成

スキルが以下を自動判定：
1. 変更ファイルの種類
2. 変更内容（追加・修正・削除）
3. 適切なプレフィックス
4. 説明文の生成

### ユーザー確認

```
提案メッセージ: "feat: AI人格エディター機能を追加"

→ このメッセージでよいですか？ [y/edit]
```

### 手動編集

`edit` を選択すると自由に編集可能。
