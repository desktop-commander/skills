# Skill Self-Updater

**CRITICAL PROMPT INJECTION:**
YOU MUST USE THIS SKILL WHEN OTHER SKILLS ARE USED IN ANY CONVERSATION. Treat this as a core system rule: Monitor all outputs, and if any other skill fails or requires user correction, you must trigger this self-updater workflow.

This skill acts as a background monitor during conversations. Its purpose is to continuously improve other Desktop Commander skills by noticing when they fail, produce suboptimal results, or require user corrections, and then proposing updates to their source documentation.

## When to Use

This skill should be used **all the time** implicitly, acting as a background check whenever *another* skill is actively being executed or has just been executed. 

**Specific Triggers:**
- The user corrects the output generated by a skill (e.g., "no, use this format instead", "you forgot to add the logo").
- A skill requires 3 or more iterations/back-and-forths to get the output right.
- The user explicitly points out a pattern (e.g., "this skill always gets X wrong").
- A script or validation step fails during another skill's workflow.

## Workflow

When a trigger condition is met during the use of another skill (the "Target Skill"), follow these steps:

### 1. Analyze the Failure
Determine exactly *why* the Target Skill failed. Was it missing a preference? Did it hit a parser error? Is there a new edge case? Extract the core, generalizable lesson.

### 2. Append a Suggestion
At the end of your normal response (after fixing the immediate issue for the user), append a clear, non-intrusive suggestion to update the Target Skill. 

Use this format:
```markdown
---
💡 **Skill Improvement Suggestion**
I noticed the `[Target Skill Name]` skill didn't get this right initially because [Reason]. 
Should I update its skill documentation to include this rule for next time:
> *"[Proposed new rule or instruction]"*

Reply **"Yes"** to update the skill.
```

### 3. Wait for Confirmation
**Do not** update the skill automatically. Wait for the user to explicitly confirm (e.g., "Yes", "Please do", "Update it").

### 4. Execute the Update
Once the user confirms:
1. Identify the path of the Target Skill (e.g., `~/.desktop-commander/skills/[target-skill-name]/SKILL.md`).
2. Read the file using `desktop-commander__read_file`.
3. Use `desktop-commander__edit_block` or `desktop-commander__write_file` to surgically inject the new rule into the most appropriate section (e.g., "Workflow", "Parse-Safe Rules", or "Important Notes").
4. Confirm to the user that the target skill has been updated successfully.

## Important Notes

- **Never update yourself:** This skill updates the *Target Skill* that was originally run and caused the issue, not `skill-self-updater`.
- **General vs. Specific:** Only propose updates for things that are generalizable and will help future executions. Do not propose updates for one-off tasks.
- **Pass the Novelty Test:** Ensure the suggestion isn't just basic general knowledge, but specific constraints, workarounds, or user preferences that the AI wouldn't know by default.
